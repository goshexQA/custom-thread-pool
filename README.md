# Курсовая работа: Кастомный пул потоков (Thread Pool)

## 1. Архитектура решения

### Основные компоненты:
- **CustomExecutor** — интерфейс, расширяющий Executor с методами submit, shutdown и awaitTermination
- **CustomThreadPoolExecutor** — основная реализация пула с Builder для настройки
- **Worker** — рабочий поток, обрабатывающий задачи из очереди
- **CustomThreadFactory** — фабрика для создания именованных потоков с логированием
- **RejectedTaskHandler / AbortPolicy** — политика обработки переполнения очереди

### Принцип работы:
1. Пул создается с фиксированным количеством потоков (corePoolSize)
2. Задачи помещаются в общую очередь BlockingQueue
3. Рабочие потоки забирают задачи из очереди и выполняют их
4. При переполнении очереди срабатывает политика отказа (AbortPolicy)

## 2. Анализ производительности

### Сравнение с ThreadPoolExecutor из стандартной библиотеки

| Характеристика | Стандартный ThreadPoolExecutor | Кастомный пул |
|----------------|-------------------------------|---------------|
| Гибкость настройки | Высокая (corePoolSize, maxPoolSize, keepAliveTime) | Средняя (только corePoolSize, queueSize) |
| Очереди | Несколько видов (ArrayBlockingQueue, LinkedBlockingQueue) | Одна реализация (LinkedBlockingQueue) |
| Политики отказа | 4 встроенных (Abort, CallerRuns, Discard, DiscardOldest) | 1 реализованная (Abort) |
| Производительность | Оптимизирована годами | Базовая реализация |

4. Механизм распределения задач
В текущей реализации используется единая очередь задач (одна BlockingQueue для всех воркеров). Это простейший подход с автоматической балансировкой:

Все рабочие потоки конкурируют за задачи из одной очереди

Свободный поток забирает следующую задачу (FIFO)

Автоматическая балансировка за счет того, что простаивающие потоки сразу получают новые задачи

Преимущества подхода:
Простота реализации

Отсутствие "голодания" потоков

FIFO порядок выполнения

Недостатки:
Конкуренция за общую очередь

Нет приоритезации задач

Одна точка синхронизации

Возможные улучшения:
Реализовать несколько очередей (одна на поток) с балансировкой Round Robin

Добавить приоритеты задач

Реализовать work stealing (воровство задач из чужих очередей)
